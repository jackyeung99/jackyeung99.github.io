Efficient Inference of Rankings From Multi-Body Comparisons [In Review]

This paper introduces a generalization of the efficient iterative step developed by Mark Newman for the Bradley-Terry (BT) model, extending its application from two-entity interactions to multi-body interactions(Plackett-Luce). The BT probabilistic model, similar to the Elo rating system used in chess, is an algorithm for computing player strength scores .

Two-Entity vs. Multi-Body Interactions

Two-entity games involve head-to-head interactions between two teams, players, or entities, such as soccer matches or boxing matches. In contrast, multi-body interactions encompass scenarios involving more than two entities, such as Formula One racing or tournaments.

Extending the Bradley-Terry Model

To accommodate these higher-order interactions, we employ hypergraphs, a generalization of standard graphs, aswell as an expansion of the likelihood function as the framework of adapting the efficient schema of the BT model for multi-body interactions. This extension allows for quicker estimation of player strength scores in complex multi-entity competitions.

The experiments were conducted on both real preference data and synthetically generated datasets, enabling a more comprehensive analysis of the capabilites of this model.

Distributed Matrix Multiplication With Map-Reduce and GCP(Google Cloud Platform)

With the massive rise in the popularity of Machine Learning, the need for efficient computation has become paramount. In the Engineering Cloud Computing class, we were tasked with leveraging Google's Map-Reduce architecture to distribute workloads across multiple virtual machines.

Project Overview

To explore distributed computing efficiency, I first identified and recreated a research paper that analyzed the performance of different matrix multiplication schemas. This involved designing an orchestration system to efficiently partition matrices into smaller chunks.

Implementation on Google Cloud Platform

Each matrix was appropriately split into chunks, ensuring that each pair of submatrices was distributed to Google Cloud Platform's (GCP) serverless functions for parallel computation. After processing, the results were aggregated to reconstruct the final matrix, demonstrating the scalability and efficiency of serverless architectures in large-scale computations. This scheme was on average 8x faster than the standard matrix multiplication method using GCP.

Group Interactions of Professional Golf Players

Through the use of Network Science, I modeled the performance of PGA (Professional Golf Association) players when grouped with others. The intuition for this project came from the "Tiger Effect," a phenomenon in which the large crowds following Tiger Woods caused his playing partners to struggle as a result. This project was conducted using a combination of datasets—one provided by DataGolf , a website that explores data science in professional golf, and another obtained through web scraping. I used the Selenium library to simulate human interactions, specifically iterating through a dropdown menu containing different tournament years to gather relevant information. The collected data was then analyzed using statistical methods to model the relative performance changes when players were paired with different partners. The result was a graph modeling the interactions between different player pairings.
